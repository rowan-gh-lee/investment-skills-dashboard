# 04 · 인사이트 생성 규칙 Skills.md

본 문서는 대시보드 상단 "Executive Summary"와 하단 "핵심 인사이트 · 리스크 플래그 · 실행 제안" 패널의 **문장 생성 규칙**을 정의합니다. 인사이트는 전부 **KPI → 규칙 → 문장** 결정론적 변환으로, 동일 데이터에 항상 동일 문장을 생성합니다.

## 생성 원칙

1. **숫자 근거 없이는 문장을 쓰지 않는다.** 모든 주장은 01에서 계산한 KPI를 직접 인용.
2. **결정론적 규칙만 사용.** LLM 추론 없이 if-then 로직으로만 문장을 만든다.
3. **플래그는 임계치 초과 시에만 등장.** 문제가 없으면 조용히 침묵.
4. **Action 섹션은 항상 최소 1개.** 문제 없으면 "현 구성 유지" 문장 기본값.
5. **1문장 = 1인사이트.** 줄거리 나열 금지, 길면 쪼갠다.

## 섹션 구조

생성된 인사이트는 아래 순서로 배열됩니다:

| 슬롯 | 필수 | 내용 | 아이콘 |
|---|---|---|---|
| 성과 요약 | ✅ | 포트 CAGR vs 벤치 CAGR, α, β, IR | 📈 |
| 리스크 요약 | ✅ | MDD 구간, Calmar, Sharpe/Sortino, VaR/CVaR | 🛡 |
| 구성 요약 | ✅ | HHI, 최대 비중 종목·섹터 | 🧩 |
| 스트레스 최악 | ✅ | 시나리오 최악값, 1일 손익 | 🌪 |
| 리스크 플래그 | 조건부 | 임계치 초과 시 각 한 줄 | ⚠︎ |
| 실행 제안 | ✅ (≥1) | 규칙 기반 구체적 액션 | ✅ |

## 규칙 템플릿

### 1. 성과 요약 (벤치 유무로 분기)

```
IF benchmark:
  "📈 성과: CAGR {p_cagr:+.2f}% 로 벤치마크 {bench} ({b_cagr:+.2f}%) 대비 
   {|diff|:.2f}%p {'상회' if diff>=0 else '하회'}. β {beta:.2f} · α {alpha:+.2f}%p · IR {ir:.2f}."
ELSE:
  "📈 성과: CAGR {p_cagr:+.2f}%, 연환산 변동성 {vol:.2f}%."
```

### 2. 리스크 요약

```
"🛡 리스크: MDD {mdd:+.2f}% ({peak} → {trough}) · Calmar {calmar:.2f} · 
 Sharpe {sharpe:.2f} / Sortino {sortino:.2f} · VaR95 {var95:.2f}% / CVaR95 {cvar95:.2f}%."
```

### 3. 구성 요약

```
"🧩 구성: HHI {hhi:.3f} · 최대 비중 종목 {top_ticker} ({top_w:.1f}%) · 
 1위 섹터 {top_sector} {top_sector_w:.1f}%."
```

### 4. 스트레스 최악

```
worst = min(stress, key=lambda s: s['pnl_pct'])
"🌪 스트레스 최악: '{worst.scenario}' 적용 시 1일 포트 {worst.pnl_pct:+.2f}% — 
 대비 자산(금/국채/현금) 비중 점검."
```

## 리스크 플래그 임계치

| 플래그 | 조건 | 문장 |
|---|---|---|
| MDD 과대 | `mdd < -0.20` | ⚠︎ MDD 20% 초과 — 꼬리위험 헤지 또는 현금비중 재검토 권장. |
| Sharpe 저조 | `sharpe < 0.50` | ⚠︎ Sharpe 0.5 미만 — 위험 대비 수익 효율이 낮음. 종목/섹터 선택 재검토. |
| 집중 위험 | `hhi > 0.15` | ⚠︎ HHI 0.15 초과 — 상위 몇 종목이 전체 리스크를 좌우하는 구조. |
| 과도한 변동성 | `p_vol > 1.3 × bench_vol` | ⚠︎ 포트 변동성이 벤치 대비 30% 이상 — 저베타 자산 편입 검토. |
| 상승 미포착 | `up_capture < 0.45` | ⚠︎ 일간 상승 포착률 45% 미만 — 벤치 대비 강세장에서 뒤처짐. |

임계치는 이 문서가 SSOT. 변경 시 `scripts/run_all.py::build_insights`와 동시 반영.

## 실행 제안 (Action) 규칙

| 트리거 | 제안 문장 |
|---|---|
| `mdd < -0.20` | ✅ 하방 헤지 — 풋옵션 혹은 역상관 자산(미국채/금) 5~10% 편입. |
| `hhi > 0.15` | ✅ 집중도 완화 — 상위 2개 종목 비중 각 2%p 하향 → 동종 ETF로 대체. |
| `p_vol > 1.3 × bench_vol` | ✅ 변동성 축소 — 필수소비재/헬스케어 저베타 섹터 가점. |
| 위 3개 모두 미해당 | ✅ 현 구성 유지 — 지표가 균형 범위. 분기 리밸런싱(±2%p 이탈 시)만 유지. |

## 문장 스타일

- 기본은 **한국어, 종결어미 생략 (보고서 스타일)**.
- 숫자는 퍼센트 2자리, 비율 2자리.
- 볼드·이모지·밑줄 등 강조는 HTML 태그(`<b>`)로만 (MD 리포트에선 strip).
- 길이: 각 인사이트 ≤ 90자 권장. 넘치면 2문장으로 분할.

## 예시 (실제 생성 결과)

```
📈 성과: CAGR +18.93% 로 벤치마크 SPY (+16.20%) 대비 2.73%p 상회. 
   β 0.82 · α +4.05%p · IR 0.38.

🛡 리스크: MDD -17.86% (2023-05-16 → 2024-01-23) · Calmar 1.06 · 
   Sharpe 0.80 / Sortino 1.18 · VaR95 1.55% / CVaR95 2.21%.

🧩 구성: HHI 0.058 · 최대 비중 종목 AAPL (8.0%) · 1위 섹터 IT 34.0%.

🌪 스트레스 최악: 'COVID-19 쇼크 (2020-03-16)' 적용 시 1일 포트 -10.20% — 
   대비 자산(금/국채/현금) 비중 점검.

✅ 현 구성 유지 — 지표가 균형 범위. 분기 리밸런싱(±2%p 이탈 시)만 유지.
```

## 금지 사항

❌ **금지**
- 주관적 감정 표현 ("좋다", "나쁘다", "훌륭하다").
- 출처 없는 숫자 인용.
- 시장 방향 예측 ("향후 상승 전망").
- 특정 종목 매매 추천 ("AAPL 매수 권장").

✅ **허용**
- 리스크 노출 수준 평가 ("20% 초과 — 재검토 권장").
- 포트폴리오 구조적 관찰 ("집중도 높음").
- 규칙 기반 액션 권고 ("헤지 5~10% 편입 검토").

## 새 인사이트 규칙 추가 절차

1. 새 KPI가 필요하면 `01_analysis_rules.md`에 먼저 추가.
2. 본 문서의 "규칙 템플릿" 또는 "임계치 표"에 행 추가.
3. `scripts/run_all.py::build_insights`에 분기 추가.
4. 예시 출력 1개 이상 본 문서 말미에 추가.
